1. Joins

A join is used to combine rows from two or more tables based on a related column. Joins are extremely important because most real databases store data in separate normalized tables. Instead of keeping employee and department information in one large table, a relational design stores them separately and connects them through keys.

Why joins are used

  • To fetch related data from multiple tables in one query.
  • To reduce redundancy by keeping data normalized.
  • To create meaningful reports and business insights.

Common join types

  • INNER JOIN: Returns only matching rows from both tables.
  • LEFT JOIN: Returns all rows from the left table and matching rows from the right table.
  • RIGHT JOIN: Returns all rows from the right table and matching rows from the left table.
  • FULL OUTER JOIN: Returns matching rows and non-matching rows from both tables.
  • SELF JOIN: A table is joined with itself.
  • CROSS JOIN: Returns the Cartesian product of both tables.

Example

SELECT e.employee_name, d.department_name
FROM Employees e
INNER JOIN Departments d
ON e.department_id = d.department_id;

In exams, a common trick is to ask which join returns unmatched rows. Remember: LEFT JOIN keeps everything from the left table, while INNER JOIN keeps only common data.

2. Aggregation and Grouping

Aggregation means performing calculations on multiple rows and returning a single summarized value. SQL aggregate functions are used in reporting, analytics, dashboards, and business summaries.

Main aggregate functions

  • COUNT() counts rows.
  • SUM() adds numeric values.
  • AVG() finds the average.
  • MIN() finds the smallest value.
  • MAX() finds the largest value.

GROUP BY

GROUP BY groups rows that have the same values in one or more columns. Aggregate functions are then applied to each group. This is useful when you want summaries by department, city, category, branch, or employee role. 

SELECT department_id, COUNT(*) AS total_employees, AVG(salary) AS avg_salary
FROM Employees
GROUP BY department_id;

Important rule

In a grouped query, every selected column must either be part of the GROUP BY clause or be used inside an aggregate function.

HAVING

HAVING filters groups after aggregation. It is different from WHERE, which filters rows before grouping. 

SELECT department_id, COUNT(*) AS total_employees
FROM Employees
GROUP BY department_id
HAVING COUNT(*) > 5;

3. Filtering

Filtering is the process of selecting only the required rows from a table. This is usually done using the WHERE clause. Proper filtering improves accuracy and can also improve performance by reducing unnecessary data processing.

Common filtering operators

  • =, >, <, >=, <=, <> for comparisons.
  • AND, OR, NOT for logical combinations.
  • BETWEEN for ranges.
  • IN for matching a list of values.
  • LIKE for pattern matching.
  • IS NULL and IS NOT NULL for null checking.

Examples

SELECT * FROM Employees
WHERE salary > 50000;

SELECT * FROM Employees
WHERE department_id IN (1, 2, 3);

SELECT * FROM Employees
WHERE employee_name LIKE 'A%';

Exam points

  • WHERE cannot directly use aggregate functions like COUNT().
  • HAVING is used when filtering grouped results.
  • NULL cannot be checked using = NULL; use IS NULL.

4. Set Operators

Set operators combine the results of two or more SELECT statements. The participating queries must generally have the same number of columns and compatible data types.

  • UNION: Combines results and removes duplicates.
  • UNION ALL: Combines results and keeps duplicates.
  • INTERSECT: Returns only common rows from both queries.
  • EXCEPT / MINUS: Returns rows from the first query that are not in the second.
SELECT city FROM Customers
UNION
SELECT city FROM Suppliers;

Difference to remember

UNION removes duplicates, while UNION ALL is faster because it does not perform duplicate elimination.

5. Ranking Functions

Ranking functions are window functions used to assign positions to rows within a result set. They are commonly used in leaderboards, salary ranking, top-N analysis, and branch-wise comparisons.

Important ranking functions

  • ROW_NUMBER() gives unique sequential numbers.
  • RANK() gives same rank to ties and skips the next rank.
  • DENSE_RANK() gives same rank to ties but does not skip numbers.
  • NTILE(n) divides rows into n groups.
SELECT employee_name, salary,
       RANK() OVER (ORDER BY salary DESC) AS salary_rank
FROM Employees;

If two employees have the same highest salary, both may get rank 1. In RANK() the next rank becomes 3, but in DENSE_RANK() the next rank becomes 2.

6. Normalization

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It breaks large tables into smaller related tables and defines relationships using keys.

Benefits

  • Reduces duplicate data.
  • Prevents update, insert, and delete anomalies.
  • Improves consistency and maintainability.

Normal forms

  • 1NF: Table has atomic values and no repeating groups.
  • 2NF: Table is in 1NF and all non-key attributes fully depend on the whole primary key.
  • 3NF: Table is in 2NF and has no transitive dependency.
  • BCNF: Stronger version of 3NF where every determinant is a candidate key.

Common anomalies

  • Insert anomaly: Cannot insert data without unrelated data.
  • Update anomaly: Same data must be changed in multiple places.
  • Delete anomaly: Deleting one row removes useful unrelated information.

7. Indexing

An index is a data structure that improves the speed of data retrieval operations on a table. It works like the index of a book. Instead of scanning every row, the database can directly locate the needed records more efficiently.

Advantages

  • Speeds up SELECT queries.
  • Improves joins and filtering on indexed columns.
  • Helps sorting and grouping in many cases.

Disadvantages

  • Uses extra storage space.
  • Can slow down INSERT, UPDATE, and DELETE operations because the index also needs updating.

Types of indexing

  • Primary index: Built on primary key.
  • Unique index: Ensures unique values.
  • Composite index: Built on multiple columns.
  • Clustered index: Physically arranges data rows according to the index key.
  • Non-clustered index: Separate structure that points to data rows.

8. Keys

Keys are attributes or combinations of attributes used to identify records uniquely and establish relationships between tables.

  • Primary Key: Uniquely identifies each row and cannot contain null values.
  • Candidate Key: Any attribute that can uniquely identify a row.
  • Super Key: A set of one or more attributes that uniquely identifies a row.
  • Alternate Key: Candidate keys not chosen as the primary key.
  • Foreign Key: A column that refers to the primary key of another table.
  • Composite Key: A key formed by combining two or more columns.

Key difference

Every primary key is a candidate key, but not every candidate key becomes the primary key. A foreign key may contain duplicate values because many rows can refer to the same parent record.

9. SQL vs NoSQL

SQL and NoSQL are two major database categories. SQL databases are relational and use structured tables. NoSQL databases are designed for flexibility, scalability, and unstructured or semi-structured data.

SQL Databases

  • Relational model with tables, rows, and columns.
  • Uses structured schema.
  • Supports ACID properties strongly.
  • Best for complex queries and transactional systems.
  • Examples: MySQL, PostgreSQL, Oracle, SQL Server.

NoSQL Databases

  • Non-relational model.
  • Schema can be flexible.
  • Designed for horizontal scaling.
  • Good for large-scale distributed systems and rapidly changing data.
  • Examples: MongoDB, Cassandra, Redis, CouchDB.

SQL is preferred when data consistency and relationships are critical. NoSQL is often preferred when flexibility, speed, and large-scale distributed storage are required.

10. Transactions

A transaction is a sequence of database operations treated as a single logical unit of work. Either all operations succeed or none of them are permanently applied.

ACID properties

  • Atomicity: All operations happen completely or not at all.
  • Consistency: The database moves from one valid state to another.
  • Isolation: Concurrent transactions do not interfere incorrectly.
  • Durability: Once committed, data remains saved even after system failure.
BEGIN TRANSACTION;
UPDATE Accounts SET balance = balance - 1000 WHERE account_id = 1;
UPDATE Accounts SET balance = balance + 1000 WHERE account_id = 2;
COMMIT;

If any step fails, the database can perform a ROLLBACK to restore the previous state. Transactions are especially important in banking, payment, inventory, and order-processing systems.

11. Database Objects

Database objects are logical structures used to store, manage, and manipulate data. Understanding them is important for interviews and MCQ-based exams.

  • Table: Stores data in rows and columns.
  • View: A virtual table based on a query.
  • Index: Improves data retrieval speed.
  • Sequence: Generates numeric values, often for IDs.
  • Synonym: Another name for a database object.
  • Stored Procedure: Precompiled SQL block that performs operations.
  • Function: Similar to procedure but generally returns a value.
  • Trigger: Automatically executes on events like insert, update, or delete.
  • Cursor: Used to process query results row by row.

Quick distinction

A view stores a query definition, not the actual table data in most cases. A trigger fires automatically, while a procedure must be called explicitly.

12. Quick Revision Tips

  • Remember the difference between WHERE and HAVING.
  • Know which join returns matched versus unmatched rows.
  • Understand primary key, foreign key, and candidate key clearly.
  • Revise ACID properties carefully because they are frequently asked.
  • Be comfortable with normalization up to 3NF.
  • Memorize the difference between RANK(), DENSE_RANK(), and ROW_NUMBER().
  • Review database objects like views, triggers, procedures, and indexes.

40 MCQs for Practice

Each question includes the correct answer and a short explanation for fast revision.

1. Which SQL clause is used to filter rows before grouping?

Answer: WHERE

Explanation: WHERE filters individual rows before GROUP BY is applied.

2. Which clause is used to filter groups after aggregation?

Answer: HAVING

Explanation: HAVING works on grouped data after aggregate functions are calculated.

3. Which join returns only matching rows from both tables?

Answer: INNER JOIN

Explanation: INNER JOIN includes only rows that satisfy the join condition in both tables.

4. Which join returns all rows from the left table and matched rows from the right table?

Answer: LEFT JOIN

Explanation: Unmatched right-side rows are shown as NULL.

5. Which aggregate function is used to count rows?

Answer: COUNT()

Explanation: COUNT() returns the total number of rows or non-null values.

6. Which function is used to find the average of numeric values?

Answer: AVG()

Explanation: AVG() returns the arithmetic mean of selected values.

7. Which operator is used to search for a pattern in SQL?

Answer: LIKE

Explanation: LIKE works with wildcards such as % and _.

8. Which keyword is used to remove duplicate rows when combining query results?

Answer: UNION

Explanation: UNION removes duplicates, unlike UNION ALL.

9. Which set operator keeps duplicates?

Answer: UNION ALL

Explanation: UNION ALL simply concatenates results.

10. Which ranking function skips rank numbers when ties occur?

Answer: RANK()

Explanation: If two rows tie at rank 1, the next rank becomes 3.

11. Which ranking function does not skip numbers after ties?

Answer: DENSE_RANK()

Explanation: It assigns consecutive rank values even when ties exist.

12. Which function gives a unique sequential number to each row?

Answer: ROW_NUMBER()

Explanation: Every row gets a distinct position number.

13. Which normal form removes repeating groups and ensures atomic values?

Answer: First Normal Form (1NF)

Explanation: 1NF requires each field to contain only one value.

14. Which normal form removes transitive dependency?

Answer: Third Normal Form (3NF)

Explanation: In 3NF, non-key attributes depend only on the key.

15. Which key uniquely identifies each row in a table?

Answer: Primary Key

Explanation: A primary key must be unique and cannot be NULL.

16. Which key refers to a primary key in another table?

Answer: Foreign Key

Explanation: A foreign key establishes referential integrity between tables.

17. Which key is a combination of multiple columns?

Answer: Composite Key

Explanation: Two or more attributes together uniquely identify a row.

18. Which database property ensures all or nothing execution?

Answer: Atomicity

Explanation: Atomicity guarantees a transaction fully completes or fully fails.

19. Which ACID property ensures committed data is permanently saved?

Answer: Durability

Explanation: Data remains preserved even after power failure or crash.

20. Which ACID property ensures one valid database state changes to another valid state?

Answer: Consistency

Explanation: Rules and constraints remain satisfied after a transaction.

21. Which command is used to undo changes in a transaction?

Answer: ROLLBACK

Explanation: ROLLBACK reverts uncommitted changes.

22. Which command permanently saves a transaction?

Answer: COMMIT

Explanation: COMMIT makes the changes permanent.

23. Which database object is a virtual table based on a query?

Answer: View

Explanation: A view stores a query definition and presents data like a table.

24. Which database object automatically executes when data changes occur?

Answer: Trigger

Explanation: Triggers fire on events like INSERT, UPDATE, or DELETE.

25. Which object improves query retrieval speed?

Answer: Index

Explanation: Indexes help locate rows faster than a full table scan.

26. Which SQL database type uses tables with predefined schema?

Answer: Relational or SQL database

Explanation: SQL databases use structured schema and table relationships.

27. Which database type is better known for flexible schema design?

Answer: NoSQL

Explanation: NoSQL systems often allow semi-structured or unstructured data.

28. Which clause groups rows sharing the same values?

Answer: GROUP BY

Explanation: GROUP BY creates row groups for aggregate calculations.

29. Which function finds the highest value in a column?

Answer: MAX()

Explanation: MAX() returns the greatest value.

30. Which function finds the lowest value in a column?

Answer: MIN()

Explanation: MIN() returns the smallest value.

31. Which operator is used to check whether a value lies within a range?

Answer: BETWEEN

Explanation: BETWEEN checks inclusive lower and upper boundaries.

32. Which condition should be used to check missing values?

Answer: IS NULL

Explanation: NULL comparisons do not work correctly with = operator.

33. Which join produces a Cartesian product?

Answer: CROSS JOIN

Explanation: Every row of the first table combines with every row of the second table.

34. Which join can be used to compare rows within the same table?

Answer: SELF JOIN

Explanation: A table is joined with itself using aliases.

35. Which anomaly occurs when deleting a row removes other useful information?

Answer: Delete anomaly

Explanation: Poorly designed tables may lose extra data after deletion.

36. Which anomaly occurs when the same fact must be updated in many rows?

Answer: Update anomaly

Explanation: Redundancy causes inconsistent updates.

37. Which anomaly occurs when a record cannot be inserted without another unrelated value?

Answer: Insert anomaly

Explanation: This commonly appears in unnormalized designs.

38. Which SQL object is a precompiled block of code used to perform operations?

Answer: Stored Procedure

Explanation: Procedures contain SQL logic that can be executed when needed.

39. Which type of index defines the physical order of data rows?

Answer: Clustered Index

Explanation: In a clustered index, data is stored in the same order as the index key.

40. Which set operator returns only rows common to both queries?

Answer: INTERSECT

Explanation: INTERSECT gives common records between two result sets.
